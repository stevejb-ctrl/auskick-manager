<objective>
SUB-01 / B4: the sub suggester ranks on cumulative game minutes only — it has
no notion of RECENCY, so a player subbed on late in Q3 is wrongly suggested off
again early in Q4. Fix it across all three sports by deriving a per-player
"time since last came on" signal at replay time and using it as a SOFT guard in
each suggester: don't pull a player who just came on while a longer-serving
field player is available.

There is NO per-player last-sub timestamp anywhere (not in game_events, the
store, or any migration) and we add none (D-05: no migration). The signal is
derived from existing events — the same derivation F3 (Phase 12, long-press
"time since last sub") will reuse, so it lives in the replay functions
(criterion #3: shared with F3).

DECISIONS LOCKED (no CONTEXT.md this phase):
- D-06 The recency signal is `lastSubbedOnMs[playerId]` = the absolute
  game-elapsed ms at which the player MOST RECENTLY transitioned bench->field
  (via a `swap`/sub event, OR a period-break / quarter-start lineup change that
  put them on field when they were off the prior period). It PERSISTS across
  period boundaries — this is the crux of B4 (the existing per-quarter stint
  resets at quarter_start lose cross-boundary recency: replayGame zeroes
  stintStartMs at quarter start ~fairness.ts:1072; suggestLeagueSubs zeroes
  startedAt at quarter_start ~rugby_league/fairness.ts:2073-2080).
- D-07 Threshold = one rotation window: `minStintMs = subIntervalSeconds * 1000`
  (the DERIVED interval from plan 10-01 — hence this plan depends on 10-01). A
  field player is "recent" if `elapsedMs - lastSubbedOnMs[id] < minStintMs`.
- D-08 SOFT guard: partition field-in-zone off-candidates into NON-RECENT
  (preferred) and RECENT (fallback); within each partition keep the existing
  ordering (most cumulative game-minutes off first). If a zone has only recent
  players, the suggester still picks one (no deadlock). This preserves all
  current behaviour when no one is recent.
- D-09 AFL is the reference (Task 2); league + netball mirror (Task 3). Netball
  is quarter-break based, so "recent" means "came on at the previous break"
  (within the prior period) — the same lastSubbedOnMs derivation, applied to the
  quarter-break reshuffle suggester.
- D-10 Coverage is UNIT-level (Vitest), matching the existing fairness test
  suite — far more deterministic than e2e for a ranking change. No new e2e spec
  required; only update an existing live/quarter-break spec if its suggested
  ordering shifts.

Purpose: the suggester stops churning a kid who just came on — fair rotation
respects recency, not just totals.
Output: shared replay-derived `lastSubbedOnMs` (reused by F3) + a soft recency
guard in all three suggesters, with a red-first cross-sport unit regression.
</objective>
