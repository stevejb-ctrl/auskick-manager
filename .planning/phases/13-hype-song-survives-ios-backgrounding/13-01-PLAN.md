- src/lib/live/hypeSongController.ts ‚Äî a pure reducer. Suggested shape (executor
  may refine names; keep it pure and keep the post-Q1 behaviour exact):
    type SongArmState  = "idle" | "ready" | "suspended";
    type SongArmEvent  = "ready" | "play" | "hidden" | "visible"
                       | "playSucceeded" | "playFailed";
    type SongArmAction = "none" | "play" | "rearm-then-play" | "rearm";
    function reduceSongArm(
      state: SongArmState, event: SongArmEvent,
    ): { state: SongArmState; action: SongArmAction };
  Behaviour the test pins (D-06):
    - idle    + ready          -> ready,     none
    - ready   + play           -> ready,     play
    - ready   + hidden         -> suspended, none      (OS suspends on background)
    - suspended + play         -> ready,     rearm-then-play   ‚Üê the post-Q1 fix
    - suspended + visible      -> ready,     rearm     (eager re-arm on foreground)
    - ready   + playFailed     -> suspended, none      (a rejected play = session went suspended;
                                                         next play re-arms)
    - any     + playSucceeded  -> ready,     none
    - idle    + play           -> idle,      play      (nothing armed yet is fine ‚Äî backend guards)
  Pure: same input -> same output, inputs never mutated, no imports beyond types.
